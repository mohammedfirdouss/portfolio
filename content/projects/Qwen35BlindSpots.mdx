Qwen3.5-4B Base Blind Spots is a small but targeted dataset that captures where a base model looks fluent but still fails on core reasoning tasks.

Each example includes the exact prompt, expected answer, model output, and a short explanation of why the failure happens. The goal is to turn vague model-quality concerns into concrete, reproducible probes.

## What it is

A focused LLM evaluation dataset with:

- **12 Targeted Cases:** One probe per failure category for fast, interpretable checks.
- **Grounded Annotations:** Expected outputs plus concise reasoning for each failure.
- **Cross-Category Coverage:** Arithmetic, logic, Bayesian reasoning, Winograd schema, and more.
- **Reproducible Setup:** Deterministic generation settings for consistent model comparisons.

## How It's Built

- **Model:** `Qwen/Qwen3.5-4B-Base` loaded via Hugging Face Transformers.
- **Execution:** Deterministic decoding (`do_sample=False`) to reduce variance.
- **Schema:** `id`, `category`, `prompt`, `expected_output`, `model_output`, `explanation`.
- **Packaging:** Published as a Hugging Face dataset for easy reuse and extension.
- **Analysis:** Includes practical recommendations for tuning data by error type.

## What I Learned

- **Small Benchmarks Can Be High Signal:** A compact set can quickly surface recurring blind spots.
- **Failure Taxonomy Matters:** Grouping by reasoning type makes remediation more actionable.
- **Determinism Helps Debugging:** Stable outputs make before/after fine-tuning comparisons clearer.
- **Data Strategy Beats Guesswork:** Mapping each failure mode to specific training data improves iteration.
