K8s Health AI is an operator-style project for Kubernetes troubleshooting. It watches failing workloads, gathers high-value runtime context, calls a configurable LLM backend, and writes actionable diagnosis output into a `ClusterDiagnosis` custom resource.

Instead of manually collecting pod events, specs, and logs under pressure, the system centralizes that workflow and returns concise guidance tied to specific failure classes.

## What it is

A Kubernetes diagnostics assistant with:

- **Failure Detection:** Watches for common pod failure states and classifies root patterns.
- **Context Collection:** Pulls events, pod spec details, and logs automatically.
- **Diagnosis CRD:** Writes findings into `ClusterDiagnosis` objects for team visibility.
- **Provider Flexibility:** Supports mock and multiple cloud/local LLM providers.

## How It's Built

- **Operator Runtime:** Go + controller-runtime for reconciliation and event handling.
- **Custom Resource:** `ClusterDiagnosis` API to persist diagnosis and status.
- **Collection Layer:** Internal modules for logs, events, and workload metadata.
- **LLM Layer:** Pluggable adapters (OpenAI, Bedrock, Vertex, Azure OpenAI, Ollama, mock).
- **Operations:** Prometheus metric `diagnoses_total` and `diagctl` CLI for quick inspection.

## What I Learned

- **Kubernetes Controller Patterns:** Building reliable reconciliation loops around noisy cluster events.
- **AI in Ops Tooling:** Using LLMs where they help most while preserving deterministic context collection.
- **Production Diagnostics:** Designing output that is short, useful, and connected to real runtime evidence.
- **Extensible Interfaces:** Abstracting providers without complicating the core diagnosis pipeline.
