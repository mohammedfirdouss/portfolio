This project implements a serverless platform for deploying and managing AI agents with real-time capabilities. The architecture uses event-driven patterns with EventBridge, Step Functions for workflow orchestration, and WebSocket APIs for live status updates.

Built with Python and AWS CDK, the infrastructure supports multi-agent collaboration, human-in-the-loop approvals, and integration with OpenAI and Anthropic LLMs.

## What it is

An event-driven AI agent platform featuring:

- **Agent Management:** CRUD operations for AI agents with DynamoDB persistence.
- **Task Processing:** Single-agent and multi-agent task execution with Step Functions orchestration.
- **Real-Time Updates:** WebSocket API for live task progress and status updates.
- **Human-in-the-Loop:** Approval workflows with wait-for-task-token pattern.
- **LLM Integration:** Unified interface for OpenAI and Anthropic with retry and circuit breaker patterns.

## Key Technical Details

- **AWS CDK:** Infrastructure as code with Foundation, Database, Events, Auth, and API stacks.
- **EventBridge:** Event-driven architecture for decoupled, scalable workflows.
- **Step Functions:** State machines for task execution, multi-agent collaboration, and approval flows.
- **DynamoDB:** Tables for agents, tasks, context, and WebSocket connections.
- **API Gateway:** REST API for agents/tasks and WebSocket API for real-time updates.
- **ElastiCache Redis:** Caching layer for performance.

## What I Learned

- **Event-Driven Architecture:** Designing serverless systems with EventBridge for loose coupling.
- **Step Functions Patterns:** Orchestrating complex workflows including parallel execution and human approval.
- **WebSocket at Scale:** Managing real-time connections with API Gateway WebSocket APIs.
- **Multi-Agent Systems:** Coordinating multiple AI agents with message routing and result aggregation.
- **LLM Integration:** Building resilient integrations with retry logic and circuit breakers.
