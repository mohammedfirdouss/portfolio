Bayan is an AI-assisted preparation tool for khateebs and scholars. It helps with research-heavy parts of khutbah preparation by searching the Quran corpus semantically, checking verse references, and generating a structured outline from a topic.

The system is designed to assist, not replace. It does not write the khutbah itself. It supports faster preparation while preserving the speaker's own words, reasoning, and delivery.

## What it is

A preparation companion focused on source-grounded support:

- **Semantic Search:** Finds verses by meaning across the Quran corpus.
- **Citation Verification:** Checks pasted text against source references with confidence scoring.
- **Outline Support:** Builds a structured khutbah outline with supporting ayat.
- **Research Workflow:** Keeps discovery, verification, and structuring in one flow.

## How It's Built

- **Backend:** FastAPI with async data access for responsive API endpoints.
- **Frontend:** React + Vite interface for focused preparation workflows.
- **Data Layer:** PostgreSQL + pgvector for embedding-powered retrieval.
- **LLM Orchestration:** LLM handles reasoning and structuring; corpus retrieval remains the citation source.
- **Ingestion Pipeline:** Dedicated ingestion scripts load and normalize Quran data.

## What I Learned

- **Trust-Centered AI UX:** Building assistant behavior that supports user authority instead of replacing it.
- **RAG in Practice:** Combining vector search and structured retrieval for higher-confidence outputs.
- **Domain Constraints:** Designing guardrails for sensitive, high-trust use cases.
- **Product Scoping:** Turning a broad AI idea into a practical workflow with clear boundaries.
