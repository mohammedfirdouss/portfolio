# Cruddur

Cruddur started as my first cloud project. The brief was clear: turn it into something real that could scale in the cloud without costs skyrocketing. I re-architected the app around an API-only backend, a lightweight frontend, and AWS managed services so we get reliability and observability without a heavy ops tax.

## What it is

Cruddur is a microblogging platform with authenticated feeds, profiles, and direct messaging. The frontend is React. The backend is a small Flask API. Data lands in two purpose-fit stores: Postgres (RDS) for relational data and DynamoDB for chat-style messaging. Authentication is handled by Amazon Cognito.

## Architecture at a glance

- Frontend: React built assets served by Nginx in an ECS Fargate service behind an ALB
- Backend: Flask API in ECS Fargate, health-checked via ALB target groups
- Auth: Cognito User Pools; backend verifies Bearer tokens and maps claims to internal users
- Relational data: Amazon RDS for PostgreSQL
- Messaging: DynamoDB with a GSI for query patterns and Streams + Lambda to update message-group ordering on new messages
- Media: S3 for assets and avatars, CloudFront in front of the assets bucket, API Gateway + Lambda for presigned uploads
- Observability: CloudWatch Logs, AWS X-Ray sidecar, OpenTelemetry (Honeycomb), Rollbar
- Config/secrets: SSM Parameter Store (no secrets in images)

## Notable features

- Message groups ordered by last activity (DynamoDB Streams + Lambda keeps sort keys fresh)
- Clean auth flow end-to-end (Amplify on the client, JWT verification on the API)
- Cost-aware by design (Fargate tasks, DynamoDB provisioned capacity, S3+CloudFront for static)

## How I work on it locally

- Docker Compose brings up frontend, backend, Postgres, and DynamoDB Local
- One-liners to create/seed Postgres and DynamoDB tables/scripts
- Health checks and simple bin scripts for fast feedback loops

## What I learned

- Keeping messaging fast and cheap is about the data model as much as code
- Small Flask services on Fargate scale well when paired with good ALB health checks
- Cognito + API Gateway + Lambda is a solid pattern for secure, presigned S3 uploads
