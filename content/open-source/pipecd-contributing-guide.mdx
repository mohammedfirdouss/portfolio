I expanded PipeCD’s contributing docs to make onboarding clearer for first-time contributors.

The update added practical guidance on where to ask questions, how to set up a local dev environment, and how to structure PRs for faster review.

I shipped this across multiple docs files and aligned wording/structure in both `docs-dev` and `docs-v1.0.x`.

This reduced ambiguity in the contribution path and was merged after maintainer iterations.
